# 개인용 주식 뉴스 분석기 기획 정리 (Codex 전달용)

## 0. 프로젝트 개요

이 프로젝트는 **나 혼자 사용하는 개인용 투자 보조 시스템**이다.
목표는 단순한 주가 조회기가 아니라, 다음을 제공하는 것이다.

- 세계 뉴스와 거시 이벤트를 수집
- 그 뉴스가 어떤 **산업/기업**에 영향을 주는지 해석
- 주식 차트 및 기본 시장 데이터와 함께 표시
- AI를 이용해 **투자 이유 / 반대 논리 / 체크포인트**를 생성
- 자동매매보다는 **설명 가능한 투자 판단 보조 도구**를 지향
- **세계지도 위에 뉴스를 시각화**하여 어느 나라에서 무슨 사건이 발생했고, 어느 나라/산업에 영향을 주는지를 한눈에 파악

핵심 철학:

> 세상에서 일어난 사건을 투자 가능한 논리로 변환하는 시스템

---

## 1. 목표와 범위

### 목표
- 세계 뉴스와 경제 이벤트를 바탕으로 투자 아이디어를 정리
- 차트와 뉴스를 한 화면에서 연결
- "왜 이 종목을 봐야 하는가"를 설명
- AI를 항상 실시간으로 돌리는 것이 아니라, **필요할 때만 분석 실행**
- 결과는 채팅처럼 흘러가는 답변이 아니라 **저장된 리포트 형태**로 보여주기
- **세계지도 뷰**: 발생 국가, 발언 주체, 영향 국가/산업을 지도 위에서 한 번에 확인

### 비목표
- 초단타 자동매매
- 고빈도 실시간 매매 시스템
- 대규모 서비스형 SaaS
- 완전 자동 매수/매도 추천
- 전 종목 100% 완전 커버

## 2. 핵심 개념

이 시스템의 핵심 흐름은 다음과 같다.

```text
뉴스/거시 이벤트
→ 사건(Event) 구조화 (발생 국가, 발언 주체, 영향 국가/산업 포함)
→ 세계지도 위 핀으로 시각화
→ 산업 영향 추론
→ 관련 기업 매핑
→ 투자 논리 생성
→ UI에서 차트 + 뉴스 + 인사이트로 표시
```

예시:

```text
미국 AI 데이터센터 투자 확대
→ 발생국: 미국 / 발언주체: Microsoft CEO
→ 영향국: 한국(반도체), 대만(파운드리)
→ GPU 수요 증가 → HBM 메모리 수요 증가
→ 반도체/HBM 산업 수혜
→ SK하이닉스, 삼성전자 관심
```

또는

```text
중동 긴장 고조
→ 발생국: 이란, 이스라엘 / 발언주체: 이란 외무부
→ 영향국: 전 세계(유가), 한국(항공/정유)
→ 유가 상승 가능성
→ 항공주 주의, 정유주 관심
```

## 3. 시스템이 해야 하는 일

### 3.1 뉴스/이벤트 수집
- 세계 뉴스 수집
- 경제/정책/외교/전쟁/제재/산업 뉴스 수집
- 필요시 환율, 금리, 유가, 지수 데이터도 함께 수집

### 3.2 사건 구조화
뉴스 원문을 그대로 보여주는 것이 아니라 AI를 이용해 다음 필드로 변환:

- 사건명
- **발생 국가** (origin_country) — 지도 핀 위치 기준
- **발언 주체** (speaker) — 누가 한 발언인지 (인물명, 기관명, 국가명 등)
- **영향 국가** (affected_countries) — 경제적/외교적으로 영향을 받는 나라들
- **영향 산업** (affected_industries) — 수혜/피해 산업 구분 포함
- 긍정/부정 방향
- 영향 기간(단기/중기/장기)
- 신뢰도/확신도
- 핵심 요약

예시 출력:

```json
{
  "event": "미국의 중국 반도체 규제 강화 가능성",
  "origin_country": "US",
  "speaker": {
    "name": "미국 상무부",
    "type": "government",
    "country": "US"
  },
  "affected_countries": ["CN", "KR", "TW"],
  "industries": {
    "positive": ["Korean high-end memory", "Taiwanese foundry"],
    "negative": ["China advanced semiconductor"]
  },
  "time_horizon": "mid",
  "summary": "미국의 대중 반도체 규제가 강화될 경우 대체 공급망 기대가 커질 수 있음"
}
```

### 3.3 산업 영향 추론
AI 또는 규칙 기반으로 다음을 추론:
- 어떤 산업이 수혜인지
- 어떤 산업이 피해인지
- 직접 영향인지 간접 영향인지
- 단기 뉴스인지 구조적 변화인지

### 3.4 기업 매핑
산업별 종목맵을 통해 종목 후보를 연결:
- 직접수혜
- 간접수혜
- 리스크 노출
- 이미 주가에 반영되었는지 판단 보조

### 3.5 투자 논리 생성
각 뉴스 또는 각 종목에 대해 다음을 생성:
- 관심 이유
- 보류 이유
- 리스크
- 체크포인트
- 추가로 확인할 항목

## 4. 핵심 기술

### 4.1 뉴스 → 사건 구조화
AI API를 사용해 기사 원문 또는 기사 제목/요약을 구조화된 이벤트로 바꾼다.

필수 추출 항목:
- event_type
- origin_country (ISO 3166-1 alpha-2)
- speaker (name, type, country)
- affected_countries (ISO 코드 배열)
- sectors (positive / negative 구분)
- symbols (선택)
- sentiment/direction
- time_horizon
- summary

### 4.2 사건 → 산업 연결 규칙 엔진
AI만으로는 결과가 흔들릴 수 있으므로 **규칙표(rule base)** 를 병행한다.

예시 규칙:

```json
{
  "rule_name": "유가 상승",
  "if_keywords": ["oil", "crude", "middle east", "supply risk"],
  "positive_industries": ["Refinery", "Energy"],
  "negative_industries": ["Airline", "Logistics"],
  "notes": "유가 급등은 항공/물류 비용 부담으로 작용 가능"
}
```

또는

```json
{
  "rule_name": "미국 금리 상승",
  "if_keywords": ["fed", "rate hike", "higher rates"],
  "positive_industries": [],
  "negative_industries": ["Growth Tech", "High Valuation Stocks"],
  "notes": "고밸류 성장주는 할인율 상승에 취약"
}
```

### 4.3 산업 → 종목 지식맵
AI가 산업을 판단한 뒤, 종목 연결은 내부 데이터(JSON/DB)로 처리한다.

예시:

```json
{
  "industry": "HBM",
  "stocks": [
    {
      "symbol": "000660.KS",
      "name": "SK하이닉스",
      "relation": "direct",
      "reason": "HBM 핵심 생산"
    },
    {
      "symbol": "005930.KS",
      "name": "삼성전자",
      "relation": "direct",
      "reason": "고부가 메모리 및 HBM 관련"
    }
  ]
}
```

### 4.4 투자 논리 생성기
AI는 최종 출력 문장을 만든다. 단, 앞단에서 구조화된 데이터를 넣어줘야 한다.

예시 출력:

```text
종목: SK하이닉스
관심도: 높음

이유
1. AI 서버 투자 확대 흐름 지속
2. HBM 수요 증가 가능성
3. 고부가 메모리 수혜 연결성 높음

리스크
1. 기대감 선반영 가능성
2. 단기 급등 후 조정 리스크
3. 정책 발표가 실제 집행으로 이어지지 않을 수 있음

체크포인트
- 외국인 수급 지속 여부
- HBM 공급계약 관련 뉴스
- 다음 실적 발표
```

### 4.5 반대 논리 강제 생성
확증편향 방지를 위해 반드시 포함:
- 이 논리가 틀릴 수 있는 이유
- 이미 주가에 반영되었는가
- 실제 실적과 연결되는가
- 단기 뉴스인가 구조적 변화인가

### 4.6 세계지도 지오코딩
국가 코드를 지도 좌표로 변환하는 레이어:
- ISO 3166-1 alpha-2 코드 → 국가 중심 위/경도
- GeoJSON 기반 국가 경계 데이터 (수혜/피해 국가 색 채우기용)
- 라이브러리: **React Simple Maps** 또는 **Leaflet + react-leaflet** 권장

## 6. AI 사용 전략

AI는 **항상 실시간으로 돌지 않는다.**

### 원칙
- AI는 기본적으로 **버튼형 + 스케줄형 혼합**
- 기본 데이터(차트, 시세, 뉴스 제목)는 항상 보임
- AI 분석 결과는 저장된 **리포트** 로 보여줌
- 새 뉴스가 쌓였을 때만 재분석
- 사용자가 원할 때만 수동 재분석

### 권장 실행 방식

#### 자동 실행
- 아침 1회
- 장 시작 전 1회
- 장 마감 후 1회

#### 수동 실행
- 전체 브리핑 다시 생성
- 특정 뉴스 분석
- 특정 종목 재분석
- 특정 산업만 재분석

### 이유
- 비용 절감
- 같은 뉴스 반복 분석 방지
- UI 안정성
- "항상 생각하는 AI"보다 "정리된 리포트"에 신뢰감 부여

## 7. UI 방향

### UI 핵심 컨셉
**첫 화면 = 세계지도 + 빅뉴스 10개**

진입 즉시 "오늘 세계에서 뭔 일이 있었나"를 한눈에 파악.
정보를 한꺼번에 쏟아내지 않고, 지도와 뉴스 카드만 보여준 뒤
사용자가 원하는 것을 클릭해서 깊게 들어가는 구조.

AI 분석은 필요한 순간만 실행되어 저장된 리포트로 표시한다.

### 가장 중요한 UX 원칙
1. **첫 화면은 단순하게** — 세계지도 + 빅뉴스 10개만
2. 클릭해서 들어갈수록 정보가 깊어지는 드릴다운 구조
3. AI가 돌지 않을 때도 화면이 비어 있으면 안 됨
4. AI 결과는 채팅형보다 리포트형으로 보여줌
5. 마지막 분석 시각 / 분석 상태를 표시
6. 실시간 데이터층과 AI 해석층을 구분
7. **지도 위 핀 클릭 한 번으로 뉴스 발생지 → 영향국 → 관련 산업까지 연결**

## 8. 추천 UI 구조

### 8.1 전체 레이아웃
2단 구조 (첫 화면은 지도 + 뉴스만, 상세 진입 시 오른쪽 패널 등장)

```text
[왼쪽 사이드바]   [가운데 메인 영역]
- Home (지도+빅뉴스)  - 세계지도 + 빅뉴스 10개     ← 첫 화면
- News                - 뉴스 리스트 + 상세           ← 클릭 후 진입
- Sectors             - 산업별 분석
- Stocks              - 종목 상세
- Watchlist
- Settings
```

> 오른쪽 인사이트 패널은 뉴스/종목 상세 화면에서만 등장.

## 9. 화면 설계

### 9.1 Home (첫 화면) — 세계지도 + 빅뉴스 10개
**목적: 진입 즉시 "오늘 세계에서 무슨 일이 있었나"를 파악.**
정보 과부하 없이 지도와 뉴스 카드 10개만 보여준다.

#### 포함 요소 (이것만)
- 상단 얇은 바: 날짜 + 마지막 분석 시각 + `브리핑 생성` 버튼
- **세계지도** (화면 상단 60%)
  - 뉴스 발생국 핀
  - 수혜/피해 국가 하이라이트
  - 핀 클릭 시 미니 팝업 (제목 + 발언주체 + 영향국/산업)
- **빅뉴스 카드 10개** (화면 하단 40%, 가로 스크롤)
  - 뉴스 제목 (1줄)
  - 발생국 국기 + 발언 주체
  - 수혜/피해 태그 (산업명 1~2개)
  - 중요도 점수

#### 뉴스 카드 예시

```text
┌──────────────────────────────┐
│ 🇺🇸 Microsoft CEO            │
│ AI 데이터센터 1000억 달러 투자  │
│                               │
│ 수혜 ↑ HBM  수혜 ↑ 파운드리   │
│ ★★★★☆  2시간 전              │
└──────────────────────────────┘
```

#### 예시 레이아웃

```text
┌─────────────────────────────────────────────────┐
│ 2026-03-17  마지막 분석 08:40    [브리핑 생성]   │
├─────────────────────────────────────────────────┤
│                                                   │
│               세  계  지  도                      │
│                                                   │
│  🔵US(3)   🔵CN(2)   🔵EU(1)                     │
│  [초록] KR / TW  [빨강] CN                       │
│                                                   │
├─────────────────────────────────────────────────┤
│  빅뉴스 10                     ← 가로 스크롤     │
│  [카드1] [카드2] [카드3] [카드4] [카드5] ...     │
└─────────────────────────────────────────────────┘
```

#### 빅뉴스 선정 기준
AI가 다음 기준으로 상위 10개를 선별:
1. 글로벌 시장 파급력 (금리/환율/원자재 직접 영향)
2. 주요국 정부/중앙은행 발언
3. 대형 기업 실적/투자 발표
4. 지정학적 긴장 고조

#### 클릭 흐름
- 카드 클릭 → News 상세 화면으로 이동
- 지도 핀 클릭 → 미니 팝업 → "자세히 보기"로 News 상세 이동

---

### 9.2 World Map 화면 (지도 전용 탐색)
Home의 지도를 전체 화면으로 확장한 탐색 화면. 필터와 타임라인 중심.

#### 핵심 목적
- **어느 나라에서 발생한 뉴스인지** 지도 위 핀으로 표시
- **누가 무슨 말을 했는지** (발언 주체 + 요약) 핀 클릭 시 표시
- **어느 나라가 영향을 받는지** 수혜/피해 국가를 색으로 구분하여 하이라이트
- **어느 산업이 영향을 받는지** 우측 패널에 표시

#### 지도 표현 규칙

| 표현 요소 | 설명 |
|---|---|
| 핀 색상 (파랑) | 뉴스 발생 국가 |
| 국가 채우기 (초록) | 수혜 영향을 받는 나라 |
| 국가 채우기 (빨강) | 피해 영향을 받는 나라 |
| 핀 크기 | 뉴스 중요도 / 영향 규모 비례 |
| 핀 클러스터링 | 동일 국가 뉴스 여러 개일 때 묶음 표시 |

#### 핀 클릭 시 팝업 내용
```text
[미국] Microsoft CEO
"AI 데이터센터에 1000억 달러 투자 계획 발표"

영향 국가: 한국 (수혜), 대만 (수혜), 중국 (피해)
영향 산업: HBM 메모리 ↑, 파운드리 ↑, 중국 반도체 ↓

[관련 뉴스 보기]  [산업 분석]  [종목 찾기]
```

#### 우측 패널 (핀 선택 시)
- 뉴스 제목 + 출처 + 시간
- 발언 주체 (기관/인물/국가)
- 영향 국가 목록 (수혜 / 피해 구분)
- 영향 산업 목록 (수혜 / 피해 구분)
- AI 요약 (분석 완료된 경우)
- `이 뉴스 분석` 버튼
- `관련 종목 찾기` 버튼

#### 상단 필터 바
- 날짜 범위 (오늘 / 3일 / 1주)
- 산업 필터 (반도체 / 방산 / 에너지 등)
- 뉴스 유형 필터 (정책 / 외교 / 경제 / 기업)
- 영향 방향 필터 (수혜만 / 피해만 / 전체)

#### 예시 레이아웃

```text
┌─────────────────────────────────────────────────────────┐
│ [오늘] [3일] [1주] | 산업▼ | 유형▼ | 방향▼              │
├────────────────────────────────────────┬────────────────┤
│                                        │ 선택 뉴스 상세  │
│   세 계 지 도                           │                │
│                                        │ [미국] MS CEO  │
│   🔵 미국 (3건)    🔵 중국 (2건)        │ AI 투자 1000억 │
│                                        │                │
│   [초록] 한국 / 대만 ← 수혜 하이라이트  │ 영향국          │
│   [빨강] 중국 ← 피해 하이라이트         │ 한국 ↑ 대만 ↑  │
│                                        │ 중국 ↓          │
│   🔵 중동 (1건)                         │                │
│                                        │ 영향 산업       │
│                                        │ HBM ↑ 파운드리↑│
│                                        │                │
│                                        │ [뉴스 분석]     │
│                                        │ [종목 찾기]     │
└────────────────────────────────────────┴────────────────┘
```

#### 기술 구현 포인트
- 지도 라이브러리: **React Simple Maps** (SVG 기반, 가볍고 커스터마이즈 용이)
  - 대안: Leaflet + react-leaflet (타일 기반, 줌/패닝 강력)
- 국가 경계 데이터: GeoJSON (Natural Earth 110m 해상도 권장)
- 핀 좌표: 국가 코드 → 위/경도 변환 테이블 (JSON으로 관리)
- 수혜/피해 국가 하이라이트: GeoJSON feature의 fill 색상 동적 변경

- 2차전지
- 에너지

이유:
- 세계 뉴스와 연결이 비교적 명확
- 국가 관계/정책/원자재 영향 해석이 쉬움
- 산업 → 종목 연결맵을 만들기 좋음

## 21. 한 줄 정의

이 프로젝트는:

> 세계 뉴스와 시장 데이터를 수집하고,
> AI와 규칙 엔진을 이용해 산업/종목 영향을 해석한 뒤,
> **세계지도 위에 뉴스를 시각화하고** 차트와 함께 설명 가능한 투자 논리를 보여주는
> 개인용 주식 리서치 데스크보드

---

## 22. Codex에게 전달할 때의 핵심 요청 포인트

Codex 작업 시 요청할 방향:

1. 개인용 도구로 시작
2. React + FastAPI 기준
3. MVP는 4개 화면 구현
   - Dashboard
   - **World Map** ← 핵심 신규
   - News
   - Stock Detail
4. AI는 항상 돌지 않고, 버튼형 + 스케줄형 혼합
5. 뉴스 분석 결과는 저장형 리포트로 보여주기
6. 산업 규칙표와 종목맵은 JSON 파일로 시작
7. 섹터는 반도체 중심으로 시작
8. 코드 구조는 확장 가능하게 작성
9. **AI 뉴스 구조화 시 origin_country, speaker, affected_countries 필드 반드시 추출**
10. **세계지도는 React Simple Maps + GeoJSON 기반으로 구현**

## 23. 추가 메모

- 처음부터 "승률 높은 자동매매"를 목표로 하지 말 것
- 먼저 "내가 납득할 수 있는 투자 논리 정리 시스템"으로 시작
- AI는 답을 단정하는 도구가 아니라, **정리와 해석 보조 도구**
- 가장 중요한 것은 **일관된 규칙과 종목맵**
- UI는 실시간 AI 채팅보다 **분석 데스크보드**가 더 적합
- **세계지도는 단순 시각화가 아니라 탐색의 시작점** — 지도에서 핀을 클릭하는 것만으로 뉴스 → 영향국 → 산업 → 종목까지 연결되어야 한다

## 24. 최종 요약

이 프로젝트의 핵심은 다음 5줄로 요약된다.

```text
뉴스를 가져온다 (발생국 / 발언주체 포함)
→ 사건으로 구조화한다 (영향국 / 영향산업 추출)
→ 세계지도 위에 시각화한다 (핀 + 국가 하이라이트)
→ 산업과 종목으로 연결한다
→ 차트와 함께 투자 논리로 보여준다
```

그리고 UI/UX 원칙은 다음과 같다.

```text
기본 데이터는 항상 보인다
지도에서 세계 흐름을 한눈에 파악한다
AI는 필요할 때만 분석한다
분석 결과는 저장된 리포트로 보여준다
사용자는 지도 → 뉴스 → 종목 순으로 탐색한다
```
