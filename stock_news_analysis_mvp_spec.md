# 개인용 주식 뉴스 분석기 기획 정리 (Codex 전달용)

## 0. 프로젝트 개요

이 프로젝트는 **나 혼자 사용하는 개인용 투자 보조 시스템**이다.  
목표는 단순한 주가 조회기가 아니라, 다음을 제공하는 것이다.

- 세계 뉴스와 거시 이벤트를 수집
- 그 뉴스가 어떤 **산업/기업**에 영향을 주는지 해석
- 주식 차트 및 기본 시장 데이터와 함께 표시
- AI를 이용해 **투자 이유 / 반대 논리 / 체크포인트**를 생성
- 자동매매보다는 **설명 가능한 투자 판단 보조 도구**를 지향

핵심 철학:

> 세상에서 일어난 사건을 투자 가능한 논리로 변환하는 시스템

---

## 1. 목표와 범위

### 목표
- 세계 뉴스와 경제 이벤트를 바탕으로 투자 아이디어를 정리
- 차트와 뉴스를 한 화면에서 연결
- "왜 이 종목을 봐야 하는가"를 설명
- AI를 항상 실시간으로 돌리는 것이 아니라, **필요할 때만 분석 실행**
- 결과는 채팅처럼 흘러가는 답변이 아니라 **저장된 리포트 형태**로 보여주기

### 비목표
- 초단타 자동매매
- 고빈도 실시간 매매 시스템
- 대규모 서비스형 SaaS
- 완전 자동 매수/매도 추천
- 전 종목 100% 완전 커버

## 2. 핵심 개념

이 시스템의 핵심 흐름은 다음과 같다.

```text
뉴스/거시 이벤트
→ 사건(Event) 구조화
→ 산업 영향 추론
→ 관련 기업 매핑
→ 투자 논리 생성
→ UI에서 차트 + 뉴스 + 인사이트로 표시
```

예시:

```text
미국 AI 데이터센터 투자 확대
→ GPU 수요 증가
→ HBM 메모리 수요 증가
→ 반도체/HBM 산업 수혜
→ SK하이닉스, 삼성전자 관심
```

또는

```text
중동 긴장 고조
→ 유가 상승 가능성
→ 항공 비용 부담 / 정유 수혜 가능성
→ 항공주 주의, 정유주 관심
```

## 3. 시스템이 해야 하는 일

### 3.1 뉴스/이벤트 수집
- 세계 뉴스 수집
- 경제/정책/외교/전쟁/제재/산업 뉴스 수집
- 필요시 환율, 금리, 유가, 지수 데이터도 함께 수집

### 3.2 사건 구조화
뉴스 원문을 그대로 보여주는 것이 아니라 AI를 이용해 다음 필드로 변환:

- 사건명
- 관련 국가
- 관련 산업
- 긍정/부정 방향
- 영향 기간(단기/중기/장기)
- 신뢰도/확신도
- 핵심 요약

예시 출력:

```json
{
  "event": "미국의 중국 반도체 규제 강화 가능성",
  "countries": ["US", "CN"],
  "industries": ["Semiconductor", "HBM", "Equipment"],
  "direction": {
    "positive": ["Korean high-end memory"],
    "negative": ["China advanced semiconductor"]
  },
  "time_horizon": "mid",
  "summary": "미국의 대중 반도체 규제가 강화될 경우 대체 공급망 기대가 커질 수 있음"
}
```

### 3.3 산업 영향 추론
AI 또는 규칙 기반으로 다음을 추론:
- 어떤 산업이 수혜인지
- 어떤 산업이 피해인지
- 직접 영향인지 간접 영향인지
- 단기 뉴스인지 구조적 변화인지

### 3.4 기업 매핑
산업별 종목맵을 통해 종목 후보를 연결:
- 직접수혜
- 간접수혜
- 리스크 노출
- 이미 주가에 반영되었는지 판단 보조

### 3.5 투자 논리 생성
각 뉴스 또는 각 종목에 대해 다음을 생성:
- 관심 이유
- 보류 이유
- 리스크
- 체크포인트
- 추가로 확인할 항목

## 4. 핵심 기술

### 4.1 뉴스 → 사건 구조화
AI API를 사용해 기사 원문 또는 기사 제목/요약을 구조화된 이벤트로 바꾼다.

필수 추출 항목:
- event_type
- countries
- sectors
- symbols(선택)
- sentiment/direction
- time_horizon
- summary

### 4.2 사건 → 산업 연결 규칙 엔진
AI만으로는 결과가 흔들릴 수 있으므로 **규칙표(rule base)** 를 병행한다.

예시 규칙:

```json
{
  "rule_name": "유가 상승",
  "if_keywords": ["oil", "crude", "middle east", "supply risk"],
  "positive_industries": ["Refinery", "Energy"],
  "negative_industries": ["Airline", "Logistics"],
  "notes": "유가 급등은 항공/물류 비용 부담으로 작용 가능"
}
```

또는

```json
{
  "rule_name": "미국 금리 상승",
  "if_keywords": ["fed", "rate hike", "higher rates"],
  "positive_industries": [],
  "negative_industries": ["Growth Tech", "High Valuation Stocks"],
  "notes": "고밸류 성장주는 할인율 상승에 취약"
}
```

### 4.3 산업 → 종목 지식맵
AI가 산업을 판단한 뒤, 종목 연결은 내부 데이터(JSON/DB)로 처리한다.

예시:

```json
{
  "industry": "HBM",
  "stocks": [
    {
      "symbol": "000660.KS",
      "name": "SK하이닉스",
      "relation": "direct",
      "reason": "HBM 핵심 생산"
    },
    {
      "symbol": "005930.KS",
      "name": "삼성전자",
      "relation": "direct",
      "reason": "고부가 메모리 및 HBM 관련"
    }
  ]
}
```

### 4.4 투자 논리 생성기
AI는 최종 출력 문장을 만든다. 단, 앞단에서 구조화된 데이터를 넣어줘야 한다.

예시 출력:

```text
종목: SK하이닉스
관심도: 높음

이유
1. AI 서버 투자 확대 흐름 지속
2. HBM 수요 증가 가능성
3. 고부가 메모리 수혜 연결성 높음

리스크
1. 기대감 선반영 가능성
2. 단기 급등 후 조정 리스크
3. 정책 발표가 실제 집행으로 이어지지 않을 수 있음

체크포인트
- 외국인 수급 지속 여부
- HBM 공급계약 관련 뉴스
- 다음 실적 발표
```

### 4.5 반대 논리 강제 생성
확증편향 방지를 위해 반드시 포함:
- 이 논리가 틀릴 수 있는 이유
- 이미 주가에 반영되었는가
- 실제 실적과 연결되는가
- 단기 뉴스인가 구조적 변화인가

## 6. AI 사용 전략

AI는 **항상 실시간으로 돌지 않는다.**

### 원칙
- AI는 기본적으로 **버튼형 + 스케줄형 혼합**
- 기본 데이터(차트, 시세, 뉴스 제목)는 항상 보임
- AI 분석 결과는 저장된 **리포트** 로 보여줌
- 새 뉴스가 쌓였을 때만 재분석
- 사용자가 원할 때만 수동 재분석

### 권장 실행 방식

#### 자동 실행
- 아침 1회
- 장 시작 전 1회
- 장 마감 후 1회

#### 수동 실행
- 전체 브리핑 다시 생성
- 특정 뉴스 분석
- 특정 종목 재분석
- 특정 산업만 재분석

### 이유
- 비용 절감
- 같은 뉴스 반복 분석 방지
- UI 안정성
- "항상 생각하는 AI"보다 "정리된 리포트"에 신뢰감 부여

---

## 7. UI 방향

### UI 핵심 컨셉
**개인 투자 데스크보드 + 리서치 화면**

즉, 주가 앱처럼 기본 데이터는 늘 보이되,  
AI 분석은 필요한 순간만 실행되어 저장된 리포트로 표시한다.

### 가장 중요한 UX 원칙
1. AI가 돌지 않을 때도 화면이 비어 있으면 안 됨
2. AI 결과는 채팅형보다 리포트형으로 보여줌
3. 마지막 분석 시각 / 분석 상태를 표시
4. 새 뉴스가 쌓였으면 "재분석 권장" 배지 표시
5. 실시간 데이터층과 AI 해석층을 구분

## 9. 화면 설계

### 9.1 Dashboard
첫 화면. "오늘 뭐부터 볼지" 빠르게 파악.

#### 포함 요소
- 오늘 날짜
- 시장 상태
- 마지막 AI 분석 시각
- `오늘 브리핑 생성` 버튼
- 오늘의 핵심 뉴스 5개
- 수혜 가능 산업
- 리스크 산업
- 관심 종목 카드
- 주요 지표
  - 나스닥
  - S&P500
  - 달러/원
  - 유가
  - 금리
  - 코스피/코스닥(선택)
- 관심 종목 미니 차트

#### 예시 레이아웃

```text
┌────────────────────────────────────────────┐
│ 오늘 브리핑 | 마지막 분석 08:40 | [전체분석] │
├────────────────────────────────────────────┤
│ 핵심 뉴스            │ AI 인사이트           │
│ - 미국 금리 동결     │ 수혜 산업: 반도체      │
│ - 중국 경기 부양     │ 리스크 산업: 항공      │
│ - 유가 상승          │ 관심 종목: SK하이닉스  │
├────────────────────────────────────────────┤
│ 주요 지표                                     │
│ 나스닥 / 달러원 / 유가 / 금리 / 코스피       │
├────────────────────────────────────────────┤
│ 관심 종목 미니차트                            │
└────────────────────────────────────────────┘
```

---

### 9.2 News 화면
뉴스 중심 화면.

#### 좌측
- 뉴스 리스트
- 시간
- 출처
- 중요도
- 분석 안 됨 / 분석 완료 / 오래됨 표시

#### 우측
선택한 뉴스 상세:
- 제목
- 요약
- 사건 분류
- 관련 국가
- 영향 산업
- 긍정/부정 방향
- 관련 종목
- AI 해석
- 반대 논리

#### 버튼
- `이 뉴스 분석`
- `관련 종목 찾기`
- `투자 논리 생성`

핵심 의도:
- 뉴스는 먼저 쌓아두고
- 내가 누를 때만 AI가 도는 구조

---

### 9.3 Stock Detail 화면
종목 중심 화면.

#### 상단
- 종목명
- 현재가 / 등락률
- 최근 분석 시각
- `이 종목 다시 분석` 버튼

#### 좌측 메인
- 캔들 차트
- 거래량
- 이벤트 마커
- 수급(기관/외국인) 선택적 표시

#### 우측 인사이트
- 현재 투자 논리
- 긍정 요인 3개
- 리스크 3개
- 확인이 필요한 포인트
- 관련 뉴스 목록

#### 하단
- 산업 연결도
- 이 종목이 왜 이 산업에 묶이는지
- 과거 유사 사례(선택)

예시:

```text
┌────────────────────────────────────────────┐
│ SK하이닉스   현재가 / 등락률   [다시분석]     │
├────────────────────────────────────────────┤
│ 차트                     │ 투자 논리         │
│ 캔들 / 거래량            │ 1. HBM 수요 증가  │
│ 이벤트 마커              │ 2. AI 서버 확대   │
│                         │ 3. 실적 기대      │
│                         │                   │
│                         │ 리스크            │
│                         │ 1. 단기 과열      │
│                         │ 2. 기대 선반영    │
├────────────────────────────────────────────┤
│ 관련 뉴스 / 산업 연결 / 체크포인트            │
└────────────────────────────────────────────┘
```

---

### 9.4 Sector 화면
산업 단위 분석 화면.

예: 반도체 선택 시
- 오늘 반도체 관련 뉴스
- 산업 분위기
- 수혜 종목 / 리스크 종목
- 산업 연결도
- 공급망 연결도
- 최근 주요 이벤트

이 화면은 뉴스가 종목 하나로 바로 연결되지 않을 때 유용함.

## 11. 상태/배지 UX

AI가 항상 돌지 않으므로, 상태 표현이 매우 중요하다.

표시 예:
- 분석 안 됨
- 분석 완료
- 오래됨
- 새 뉴스 반영 필요
- 재분석 권장

예시 문구:
- `분석 완료 · 2시간 전`
- `새 뉴스 3건으로 재분석 권장`
- `AI 분석 없음`

## 13. 디자인/시각 표현

### 기본 방향
- 주식 거래소 느낌보다 **분석 도구** 느낌
- 차트/시세 데이터와 AI 인사이트를 색으로 구분

### 제안
- 배경: 다크 또는 중성톤
- 상승/하락: 시장용 색상
- AI 인사이트: 보라/청록 계열 카드
- 리스크: 노랑/주황
- 재분석 필요: 회색/오렌지 배지

### 원칙
- 시세 상승/하락 색과 AI 해석 색을 분리
- 너무 번쩍이는 UI보다 차분한 데스크보드 느낌

## 17. 사용자 흐름

### 시나리오 1: 아침에 빠르게 확인
1. 앱 실행
2. Dashboard에서 오늘 핵심 뉴스와 주요 지표 확인
3. AI 브리핑이 오래됐으면 `오늘 브리핑 생성`
4. 관심 종목 클릭 후 상세 확인

### 시나리오 2: 특정 뉴스 확인
1. News 화면에서 기사 클릭
2. 아직 분석이 없으면 `이 뉴스 분석`
3. 관련 산업/종목 확인
4. 관련 종목 상세 페이지 이동

### 시나리오 3: 특정 종목 깊게 보기
1. Watchlist 또는 Dashboard에서 종목 선택
2. 차트 + 관련 뉴스 + AI 논리 확인
3. 필요 시 `이 종목 다시 분석`

## 19. 중요한 원칙 정리

1. 뉴스 자체를 매수 신호로 사용하지 말 것
2. 산업과 종목 연결 근거가 있어야 함
3. 이미 주가에 반영되었는지 체크할 것
4. 반대 논리를 반드시 포함할 것
5. AI는 항상 돌리지 말고 리포트형으로 사용할 것
6. 처음엔 섹터를 좁혀서 시작할 것

---

## 20. 추천 초기 섹터

처음부터 전 산업을 다루지 말고 아래 중 1~3개만 선택:

- 반도체
- 방산
- 조선
- 2차전지
- 에너지

이유:
- 세계 뉴스와 연결이 비교적 명확
- 국가 관계/정책/원자재 영향 해석이 쉬움
- 산업 → 종목 연결맵을 만들기 좋음

---

## 21. 한 줄 정의

이 프로젝트는:

> 세계 뉴스와 시장 데이터를 수집하고,  
> AI와 규칙 엔진을 이용해 산업/종목 영향을 해석한 뒤,  
> 차트와 함께 설명 가능한 투자 논리를 보여주는  
> 개인용 주식 리서치 데스크보드

---

## 22. Codex에게 전달할 때의 핵심 요청 포인트

Codex 작업 시 요청할 방향:

1. 개인용 도구로 시작
2. React + FastAPI 기준
3. MVP는 3개 화면만 구현
   - Dashboard
   - News
   - Stock Detail
4. AI는 항상 돌지 않고, 버튼형 + 스케줄형 혼합
5. 뉴스 분석 결과는 저장형 리포트로 보여주기
6. 산업 규칙표와 종목맵은 JSON 파일로 시작
7. 섹터는 반도체 중심으로 시작
8. 코드 구조는 확장 가능하게 작성

## 23. 추가 메모

- 처음부터 "승률 높은 자동매매"를 목표로 하지 말 것
- 먼저 "내가 납득할 수 있는 투자 논리 정리 시스템"으로 시작
- AI는 답을 단정하는 도구가 아니라, **정리와 해석 보조 도구**
- 가장 중요한 것은 **일관된 규칙과 종목맵**
- UI는 실시간 AI 채팅보다 **분석 데스크보드**가 더 적합

## 24. 최종 요약

이 프로젝트의 핵심은 다음 4줄로 요약된다.

```text
뉴스를 가져온다
→ 사건으로 구조화한다
→ 산업과 종목으로 연결한다
→ 차트와 함께 투자 논리로 보여준다
```

그리고 UI/UX 원칙은 다음과 같다.

```text
기본 데이터는 항상 보인다
AI는 필요할 때만 분석한다
분석 결과는 저장된 리포트로 보여준다
사용자는 뉴스/종목 중심으로 탐색한다
```
